It is possible to customize your Github workflow file after it is written to the repository. Full documentation for writing Github actions files can be found [here](https://docs.github.com/en/actions/learn-github-actions/workflow-syntax-for-github-actions). This guide documents some common use-cases that you can use to set up your Github actions files.

## Multi-Process Deployments

Applications are frequently deployed with multiple process types, such as web and workers. By default, Porter assumes that each repository runs one application type.
